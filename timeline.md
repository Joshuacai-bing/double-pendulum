### GitHub使用规范（重要）

| 规范项 | 说明 |
|--------|------|
| **仓库命名** | `double-pendulum-epq` 或 `epq-physics-double-pendulum` |
| **提交频率** | 每次代码修改后立即提交，**不要到最后一次性提交**（体现工作过程） |
| **提交信息格式** | `type: description`<br>• `feat`: 新功能<br>• `fix`: 修复bug<br>• `docs`: 文档更新<br>• `refactor`: 代码重构<br>• `chore`: 杂项 |
| **必须包含的文件** | • `README.md`：项目介绍、运行方法<br>• `requirements.txt`：依赖包列表<br>• `.gitignore`：忽略除最终PDF外的编译中间文件（`.aux`, `.log`等）<br>• 所有`.py`源代码<br>• **论文相关文档（全套`.tex`源码、图片资源及最终编译好的PDF）** |
| **仓库可见性** | **务必设置为 Private（私有仓库）**，因为含有论文未经查重的核心内容，只需邀请指导老师或协作者访问 |
| **代码注释** | 关键函数和算法必须有注释，中英文均可，英文更佳 |
| **隐私保护** | 不要上传：个人密码、API密钥、大视频文件（放网盘） |

### 风险提示与应对

| 风险 | 应对措施 |
|------|---------|
